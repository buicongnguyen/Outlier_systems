Study Goal

This site collects revision material for systems software-engineering assessment work. The target skill set spans five areas: databases and storage engines, operating systems with concurrency and language runtimes, distributed systems, networking, and infrastructure. Each area pairs a definitions list and core-concept notes with a bank of practice questions and explanations in four formats: single-answer, select-all-that-apply, ordering chains, and complete-the-code snippets.

Data

Databases & Storage

B-trees vs LSM-trees, ACID and isolation levels, MVCC, write-ahead logging, indexing strategy, replication and sharding, OLTP vs OLAP, and the NoSQL families.

OS

OS, Concurrency & Runtime

Processes and threads, virtual memory and paging, syscalls, mutexes and atomics, memory ordering, deadlock, CPU caches and false sharing, epoll, and async runtimes.

Distributed

Distributed Systems

CAP and consistency models, Raft consensus, quorums, delivery guarantees and idempotency, 2PC vs sagas, fencing tokens, and retries with backoff and jitter.

Network

Networking

TCP handshakes and congestion control, TLS 1.3, DNS, HTTP/1.1 vs HTTP/2 vs HTTP/3, head-of-line blocking, L4 vs L7 load balancing, and tail latency.

Infra

Infrastructure

Containers vs VMs, Kubernetes reconciliation, deployment strategies, IaC, observability and SLOs, caching layers, and message queues.

Embedded

RTOS, Boot & Host + NPU

Deterministic scheduling, Boot ROM and secure loaders, UEFI/U-Boot, device tree, NPU firmware, PCIe/SoC discovery, DMA/IOMMU, queues, interrupts, and first inference.

Practice format

Question Drills

100 questions with explanations across five sections — single-answer, multi-select, ordering chains, and complete-the-code snippets (SQL joins, shell redirection, pthread waits, DNS records, Kubernetes probes, Dockerfile caching), including 15 senior/principal-tagged production-diagnosis scenarios — with per-section and overall scores, a show-all-explanations toggle, and a reshuffling retry button.
